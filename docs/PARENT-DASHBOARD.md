# Спецификация: Родительский дашборд (Parent Dashboard)

## 1. Цели
- Быстрый обзор прогресса ребенка по предметам и классам.
- Выявление "провалов" (слабые темы) и рекомендации к тренажеру.
- Подготовка к отчетам (родительские собрания).

## 2. Пользователи
- Родитель одного ребенка.
- Родитель нескольких детей (фильтр по ребенку).
- Учитель / классный руководитель (группа учеников).

## 3. Ключевые экраны и функции

### 3.1 Общий обзор ребенка
- Карточка: Имя, аватар, класс.
- Прогресс по предметам (Русский 4, Математика 6 и т.д.) — % выполненных заданий/тем.
- Сводка по звездам (Total, Gold/Silver) и уровням (Rank).
- Индикаторы "Зона внимания" (желтый) и "Критично" (красный).

### 3.2 Детализация по предмету
- Список "островов" (тем): статус (Не начато / В процессе / Освоено).
- Метрики: количество решенных задач, % правильных, время в тренировке.
- Список "сложных тем" (низкий % правильных, много ошибок подряд).

### 3.3 История сессий
- Лента: Дата, предмет, режим (диагностика/тренировка), результат (кол-во задач, правильные, время).

### 3.4 Настройки
- Лимит сессий/времени в день.
- Вкл/выкл анимаций, звука, "режим фокусировки".

### 3.5 Экспорт/Шэринг
- Экспорт краткого отчета в PDF/скрин/линк (позже).
- Отправка отчета учителю/второму родителю (позже).

## 4. Данные от бэкенда (API)
- `GET /api/parent/children` — список детей пользователя.
- `GET /api/parent/child/:id/summary` — сводка по ребенку (задачи, звезды, ранг).
- `GET /api/parent/child/:id/sessions` — история сессий.
- `GET /api/parent/child/:id/topics` — прогресс по островам/темам.

## 5. UX/UI требования
- Mobile-first, но комфортен на Desktop.
- Крупные элементы, читабельные таблицы.
- Цветовая кодировка: Зеленый (ок), Желтый (внимание), Красный (критично).
- Соблюдение 152-ФЗ (без лишних персональных данных).

## 6. Технические требования
- Роут: `/parent` или `/dashboard`.
- Авторизация: Email + пароль (или магические ссылки).
- API: Endpoints как в п.4.
- Кэширование: Последние данные кэшируются локально (readonly при оффлайне).
