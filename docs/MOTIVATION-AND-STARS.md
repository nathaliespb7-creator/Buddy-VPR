# Система мотивации Buddy VPR

**Версия:** 1.1  
**Дата:** февраль 2025  
**Назначение:** спецификация звёзд, уровней и прогресса для разработки и продукта.

---

## Введение

В приложении два режима: **Free** («Песочница») и **Premium** («Путь Героя»). В Free мы не знаем общего числа заданий в курсе, поэтому не показываем уровни и прогресс-бар — только счётчик звёзд и салюты. В Premium известна ёмкость курса; уровни и прогресс считаются по проценту от неё (адаптивная шкала без жёстких порогов в коде).

**Правила начисления звёзд (едины для Free и Premium):**
- **gold** — верно с первой попытки без подсказки; **silver** — верно с подсказкой или со второй попытки; **empty** — неверно (звезда не начисляется).
- **Одно задание — максимум одна звезда.** Звезда привязывается к `taskId`. При повторном прохождении того же задания обновляется только тип звезды (например silver → gold), но **totalStars не увеличивается**, если звезда за это задание уже была получена. При улучшении результата: `goldCount` увеличивается, `silverCount` уменьшается, `total` остаётся прежним. Так мы защищаем от накрутки: 10 заданий × 10 проходов не должны давать 100 звёзд.

**Диагностика:** задания в режиме диагностики не дают звёзд (или дают с пометкой «не влияют на прогресс»). Диагностика — «разведка», а не «битва».

## Часть I. Текущее состояние (до рефакторинга)

### 1. Звёзды

- **Хранение:** профиль в `Home.tsx`: `stars: StarType[]`, `totalStars: number`; ключ localStorage `buddy_profile`.
- **Начисление:** в `handleTaskComplete` при верном ответе вызывается `addStar(starType)`; при `empty` звезда не добавляется. Логика типа звезды — в `TaskCard.tsx` (выбор варианта: первая попытка без подсказки → gold; текст: `validateAnswer`, score 2→gold, 1→silver).
- **Отображение:** Header — суммарные gold/silver за всё время; ProfileSummary — «⭐ totalStars всего» и блок уровня.

### 2. Уровни (текущая реализация)

- **Файл:** `client/src/lib/levelSystem.ts`, функция `calculateLevel(totalStars)`.
- **Пороги (жёсткие):** 0–50 Искатель, 51–200 Следопыт, 201–500 Хранитель, 501–1000 Магистр, 1001–2999 Легенда, 3000+ Бадди-Герой.
- **Где показывается:** Header (бейдж Lv + иконка), ProfileSummary, LevelUpModal при повышении.

### 3. Прочие счётчики

- **Раунд:** taskProgress в Header (текущее/всего заданий), TaskProgressBar.
- **overallProgress:** из API `/api/categories/progress`, ProgressRing в десктопной шапке (прогресс по раундам сервера, не по звёздам).
- **Диагностика:** diagnosticScores (correct/total по категориям, только «с первой попытки») → PowerCard.
- **Серия 5 подряд:** correctStreak → triggerStreakReward (конфетти/звук/тост в зависимости от уровня анимации).

### 4. Карта кода (до рефакторинга)

| Элемент | Файл / место |
|--------|----------------|
| StarType, отображение звёзд в шапке | `Header.tsx` |
| Выдача gold/silver | `TaskCard.tsx` |
| addStar, профиль, handleTaskComplete | `Home.tsx` |
| Расчёт уровня | `levelSystem.ts` |
| Профиль: getStoredProfile, saveProfile | `Home.tsx` |
| Уровень в UI | `Header.tsx`, `ProfileSummary.tsx`, `LevelUpModal.tsx` |
| overallProgress | `Home.tsx` (useMemo от categoryProgressData) |
| Streak | `Home.tsx` (correctStreak, triggerStreakReward) |

## Часть II. Целевая архитектура

### 1. Режимы Free и Premium

| Аспект | Free («Песочница») | Premium («Путь Героя») |
|--------|---------------------|------------------------|
| Идентификация | Гость, без имени | Никнейм ребёнка + профиль родителя |
| Хранение | Только localStorage | Сервер (Cloud Sync) |
| Звёзды | Цифра + салют при получении | То же + рейтинг, статистика, streak |
| Уровни/ранги | Не показываем | Адаптивные по % от ёмкости курса |
| Прогресс-бар курса | Не показываем | (collectedStars / moduleCapacity) × 100% |
| Сообщение ребёнку | «Смотри, сколько звёзд ты собрал!» | «Ты прошёл X% пути к званию» |

### 2. Формат профиля

```ts
type UserTier = 'free' | 'premium';

interface UserProfile {
  avatar: AvatarChoice;
  tier: UserTier;
  nickname: string | null;   // null для Free
  stars: {
    total: number;
    gold: number;
    silver: number;
  };
  // Для пересдачи: привязка звезды к taskId. Ключ — id задания, значение — лучший полученный тип.
  // При повторном прохождении обновляем только тип (silver→gold), total не увеличиваем.
  starByTaskId?: Record<string, 'gold' | 'silver'>;
  level?: number;            // Premium: виртуальный уровень от %
  rank?: string;             // Premium: Новичок | Следопыт | Мастер | Легенда
  seenRanks?: string[];       // Premium: уже показанные ранги (чтобы не спамить LevelUpModal)
}
```

**Миграция со старого формата:** при первой загрузке из localStorage: если есть `stars: StarType[]`, вычислить `total`, `gold`, `silver` и сохранить в `stars: { total, gold, silver }`; поле `totalStars` из старого профиля отобразить в `stars.total`. Массив больше не хранить. (При отсутствии `starByTaskId` считаем, что старые данные не содержат привязки к taskId — при первой реализации пересдачи можно не мигрировать в starByTaskId, а ввести правило «звезда по taskId» только для новых прохождений.)

### 3. Ёмкость курса (moduleCapacity)

- **moduleCapacity** — это **общее число заданий во всём курсе/приложении**, а не в отдельной категории (острове). Сумма по всем доступным заданиям: `moduleCapacity = allTasks.length` или значение из API (GET `/api/metadata` → `totalTasks`).
- **MVP:** один курс на приложение, один общий capacity. Прогресс-бар в Premium отражает движение к **глобальной цели** («Путь Героя»): например, 5 островов по 100 заданий = 500 всего; ребёнок прошёл один остров (100 заданий) → 20% прогресса, а не 100%.
- В **Free** capacity не используется — ранги и прогресс-бар курса не показываем.
- В **Premium** прогресс и ранг считаются от `(stars.total / moduleCapacity) * 100`.

### 4. Ранги в Free

**Решение:** только счётчик «⭐ N звёзд» и салюты (при получении звезды и при серии 5 подряд). Без званий, без процентов, без прогресс-бара. При желании позже можно добавить вехи 10 / 50 / 100 звёзд (тост без привязки к total).

### 5. Уровни/ранги в Premium (адаптивная шкала)

- **Формула:** `progressPercent = moduleCapacity > 0 ? Math.min(100, (stars.total / moduleCapacity) * 100) : 0`.
- **Ранги по процентам:**

| progressPercent | Ранг     |
|-----------------|----------|
| 0 – &lt;20      | Новичок  |
| 20 – &lt;50     | Следопыт |
| 50 – &lt;80     | Мастер   |
| 80 – 100        | Легенда  |

- Прогресс-бар курса: ширина = `progressPercent`; подпись: «N из M звёзд» или «Осталось X% до Легенда».
- **LevelUpModal:** показывать **только в момент пересечения границы** диапазона (например, `previousPercent < 20 && currentPercent >= 20` → «Ты стал Новичком!»), а не при каждом монтировании, если ребёнок уже в ранге. Реализация: хранить в профиле `seenRanks: string[]` или сравнивать `previousPercent` с порогами; модалка срабатывает только когда `previousPercent < threshold <= currentPercent`. Иначе при входе в приложение с уже накопленным прогрессом модалка будет спамить.

### 6. Хранение и синхронизация

- **Free:** только localStorage, ключ `buddy_profile`. При отсутствии — создать профиль с `tier: 'free'`, `nickname: null`, `stars: { total: 0, gold: 0, silver: 0 }`.
- **Premium:** при изменении `stars` — PATCH `/api/user/progress` (тело: `{ stars, server_timestamp }`). При входе — GET прогресса.
- **Конфликт «офлайн vs сервер»:** нельзя слепо затирать локальный прогресс ребёнка. Пример: ребёнок набрал 20 звёзд офлайн, зашёл дома онлайн; сервер вернул 0 звёзд — если принять сервер, прогресс потеряется. **Стратегия: приоритет новизны или слияние.** Варианты: (1) `newTotal = max(serverTotal, localTotal)` и объединить gold/silver по большим значениям; (2) если `local_timestamp > server_timestamp`, принять локальные данные и отправить их на сервер. Явно прописать в коде: при синхронизации не перезаписывать локальный профиль данными с сервера, если локальные данные новее или суммарно «больше» (ребёнок мог играть офлайн). После успешного PATCH обновлять локальное состояние из ответа сервера.
- **Апгрейд Free → Premium:** при покупке/входе перенести текущие `stars` из localStorage на сервер, привязать к пользователю; показать сообщение: «Твои звёзды теперь в твоём профиле!».

### 7. Ограничения UI (честность)

- Не показывать «Уровень 5 из 10», если неизвестно общее число заданий (Free или capacity не передан).
- В Free не писать «Сохранено в облаке» и не показывать прогресс-бар курса.
- В Premium при офлайне показывать локальное состояние и при необходимости пометку «Будет синхронизировано», не обещать синхронизацию до успешного запроса.
- **Адаптивность:** в Premium на мобильных устройствах прогресс-бар курса может сворачиваться в иконку или открываться по тапу, чтобы не занимать место на экране заданий.

## Часть III. План внедрения

### Фаза 1: Профиль и звёзды

- [ ] Ввести `UserTier` и поле `tier` в профиль (по умолчанию `'free'`).
- [ ] Заменить хранение: `stars: { total, gold, silver }` вместо `stars: StarType[]` и `totalStars`; при необходимости `starByTaskId` для пересдачи.
- [ ] Миграция при загрузке: из старого `stars[]` / `totalStars` заполнить `stars.total`, `stars.gold`, `stars.silver`.
- [ ] Обновить `addStar(taskId, starType)`: если для `taskId` звезда уже есть — обновить только тип (улучшение silver→gold: gold+1, silver-1), иначе добавить новую звезду (total+1, gold или silver+1). Не увеличивать total при повторном прохождении.
- [ ] Header и все места отображения: `goldCount = profile.stars.gold`, `silverCount = profile.stars.silver`, total = `profile.stars.total`.
- [ ] **Протестировать миграцию:** на чистом localStorage со старым форматом `stars: StarType[]` (и при необходимости `totalStars`) — после загрузки профиль должен корректно превратиться в новый формат без потери данных.

### Фаза 2: Уровни и capacity (Premium)

- [ ] Источник `moduleCapacity`: `allTasks.length` или API `/api/metadata` (поле `totalTasks`); передавать в места расчёта прогресса (проп/контекст из Home).
- [ ] Реализовать `getRankAndProgress(starsTotal, moduleCapacity) → { rank, progressPercent, label }`. При отсутствии `moduleCapacity` возвращать `rank: null`, прогресс не показывать.
- [ ] Free: в ProfileSummary и Header не показывать блок уровня/ранга и прогресс-бар курса; только «⭐ N звёзд» и салюты.
- [ ] Premium: показывать ранг, прогресс-бар и подпись «N из M звёзд»; LevelUpModal — только при пересечении границы % (хранить `seenRanks` или сравнивать previous/current percent).

### Фаза 3: API и синхронизация (Premium)

- [ ] **Бэкенд (обязательно):** реализовать эндпоинт PATCH `/api/user/progress`: приём `{ stars: { total, gold, silver }, timestamp }`, сохранение по userId/session. Без него хук `useProfileSync` будет получать 404/500 при tier === 'premium'.
- [ ] При изменении звёзд в Premium: после обновления локального состояния вызывать PATCH; при ошибке — флаг `pendingSync`, повторить при следующем успешном запросе.
- [ ] При входе в Premium: GET прогресса; применять стратегию слияния/приоритета новизны (не затирать локальный прогресс, если локальные данные новее или «больше»).
- [ ] При апгрейде: отправить текущие `stars` на сервер, показать тост «Твои звёзды теперь в профиле!».

### Фаза 4: Полировка

- [ ] Тултип в шапке в режиме заданий: уточнить формулировку (не путать «верно за всё время» с «верно в раунде»).
- [ ] По желанию: в Free вехи 10/50/100 звёзд (тост без прогресс-бара).
- [ ] Проверка офлайн: Free — всё в localStorage; Premium — очередь PATCH и отображение локальных данных при отсутствии сети.
