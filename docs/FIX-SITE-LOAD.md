# Пошаговая инструкция: сайт не открывается / 404 на скрипты

Если **www.buddyvpr.ru** показывает «Загрузка…» и не переходит в приложение, а в Network видно **Pending** или **404** для `index-….js` и `index-….css`, значит браузер запрашивает файлы со старыми именами или они не отдаются. Ниже — что сделать по порядку.

---

## С мобильного открывается, с Mac (Chrome/Safari) — нет

Если с телефона **buddyvpr.ru** открывается, а с Mac — нет (белый экран, ошибка или «Не удаётся открыть»):

1. **Проверить с Mac без кэша**
   - Chrome: открой **https://www.buddyvpr.ru** → **DevTools** (Cmd+Option+I) → вкладка **Network** → включи **Disable cache** → обнови страницу (Cmd+Shift+R).
   - Safari: **Develop** → **Empty Caches**, затем обнови (Cmd+R). Если меню Develop нет: **Safari** → **Settings** → **Advanced** → включи «Show features for web developers».

2. **Проверить, что именно видит Mac**
   - В Chrome в **Network** посмотри: при загрузке **www.buddyvpr.ru** приходит ли ответ 200 для документа (первая строка)? Есть ли красные строки (404/ failed)?
   - Если документ 200, но скрипты/стили 404 или Pending — та же ситуация, что в основной инструкции ниже (редирект/кэш). Если сам документ не грузится (failed / таймаут) — возможна сетевая или DNS-разница между Mac и телефоном.

3. **Временно использовать основной URL с Mac**
   - Для работы и проверок с компьютера используй **https://buddyvpr.vercel.app** — он отдаётся тем же проектом Vercel и не зависит от домена .ru.

4. **Если с Mac открыть .ru всё же нужно**
   - Попробовать в режиме инкогнито (Chrome: Cmd+Shift+N, Safari: Private Window).
   - Попробовать другую сеть (другой Wi‑Fi или раздача с телефона).
   - В **Vercel** → проект → **Settings** → **Domains** убедиться, что для **www.buddyvpr.ru** статус **Valid**; при необходимости нажать **Refresh** и подождать 5–10 минут.

### Запасной вариант: документ грузится 5+ минут, скрипты/стили (failed)

Если в **Network** (Chrome DevTools) видно:

- **www.buddyvpr.ru** — статус **200**, но **Time 5+ мин** (запрос «висит» с Mac);
- **index-….js** и **index-….css** — статус **(failed)** или таймаут;

значит до домена .ru с этого компьютера/сети соединение очень медленное или нестабильное. Скрипты и стили не успевают подгрузиться, приложение не стартует, появляется сообщение «Загрузка занимает много времени» и кнопка «Обновить».

**Что делать:**

1. **Рабочий вариант с Mac:** открывать **https://buddyvpr.vercel.app** — тот же проект, другой хост; с компьютера обычно грузится нормально.
2. **Попробовать другую сеть:** другой Wi‑Fi, раздача с телефона — иногда маршрут до .ru меняется и сайт начинает открываться.
3. **Попробовать другой DNS на Mac:** Системные настройки → Сеть → Wi‑Fi → Дополнительно → DNS: добавить, например, `8.8.8.8` или `1.1.1.1`, применить и снова открыть **https://www.buddyvpr.ru**.

Дальше можно тестировать .ru с разных сетей и при необходимости держать vercel.app как запасной URL для десктопа.

### Обновления дизайна не видны на телефоне

Если правили мобильный UI (шапка, экран задания), а на телефоне всё как раньше — почти всегда это **кэш браузера или PWA**:

1. **Жёсткое обновление:** в браузере на телефоне обнови страницу с очисткой кэша (Safari: долгое нажатие на кнопку обновления → «Обновить без кэша»; Chrome: Меню → «Обновить» или закрой вкладку и открой сайт заново).
2. **Открыть с другого URL:** зайди на **https://buddyvpr.vercel.app** — там чаще подхватывается свежий деплой, чем на .ru.
3. **Проверить, что загружена новая сборка:** открой код страницы (в браузере на ПК: ПКМ → «Просмотр кода страницы») и поищи в `<head>` комментарий `mobile-ui:` — если его нет, грузится старая версия.
4. **Убедиться, что деплой прошёл:** Vercel → Deployments → последний коммит в статусе Ready.

## Шаг 1. Убедиться, что последний код на GitHub

1. Открой **терминал** в Cursor (меню **Terminal** → **New Terminal** или **Ctrl+`**).
2. Выполни:
   ```bash
   cd /Users/nathalie/Buddy-VPR
   git status
   ```
3. Если есть **изменённые или неотслеживаемые файлы** (например `vercel.json`, `client/`, и т.д.):
   - Добавь их в коммит и запушь:
     ```bash
     git add .
     git status
     ```
     Убедись, что в списке **нет** файла `.github-token-buddy` (он должен быть в `.gitignore`). Если он есть — сними галочку или выполни:  
     `git restore --staged .github-token-buddy`
   - Сделай коммит:
     ```bash
     git commit -m "fix: vercel rewrites и кэш для корректной загрузки assets"
     ```
   - Запушь (если раньше был 401, используй токен из файла — см. шаг 1а ниже).

**Шаг 1а. Если при `git push` снова ошибка 401 (Authentication failed):**

1. Убедись, что в корне проекта есть файл **`.github-token-buddy`** и в нём **одна строка** — твой Personal Access Token с GitHub (без пробелов и кавычек). Как создать токен — в **docs/GIT-GITHUB.md**.
2. В терминале выполни **по очереди** (подставь свой логин, если не `nathaliespb7-creator`):
   ```bash
   cd /Users/nathalie/Buddy-VPR
   git remote set-url origin "https://nathaliespb7-creator:$(cat .github-token-buddy)@github.com/nathaliespb7-creator/Buddy-VPR.git"
   git push -u origin main
   ```
3. После успешного push верни обычный URL:
   ```bash
   git remote set-url origin https://github.com/nathaliespb7-creator/Buddy-VPR.git
   ```

## Шаг 2. Дождаться нового деплоя на Vercel

1. Зайди на **https://vercel.com** и войди в аккаунт.
2. Открой проект **Buddy-VPR** (или как он у тебя назван).
3. Перейди во вкладку **Deployments**.
4. Вверху должен появиться **новый деплой** (только что запущенный после push). Статус сначала **Building**, затем должен стать **Ready** (зелёная галочка).
5. Подожди **1–3 минуты**, пока сборка закончится. Если статус **Failed** — открой деплой и посмотри логи ошибки.

---

## Шаг 3. Убедиться, что домен привязан к последнему деплою

1. В том же проекте на Vercel открой **Settings** → **Domains**.
2. Проверь, что указаны **buddyvpr.ru** и **www.buddyvpr.ru** и у них статус **Valid**.
3. Production-домен должен автоматически указывать на **последний** успешный деплой. Если есть сомнения — можно в **Deployments** открыть последний деплой со статусом **Ready** и нажать **Promote to Production** (если такая кнопка есть).

---

## Шаг 4. Жёсткое обновление страницы на buddyvpr.ru

Чтобы браузер не использовал старый закэшированный `index.html` (со старыми именами скриптов):

1. Открой в браузере **https://www.buddyvpr.ru**.
2. Открой **инструменты разработчика**: **F12** или **Cmd+Option+I** (Mac) / **Ctrl+Shift+I** (Windows), или через меню браузера: **Дополнительные инструменты** → **Инструменты разработчика**.
3. Во вкладке **Network** (Сеть) поставь галочку **Disable cache** (Отключить кэш).
4. **Не закрывая** DevTools, обнови страницу **жёстко**:
   - **Mac:** **Cmd+Shift+R**
   - **Windows:** **Ctrl+Shift+R**
   - Или: кнопка обновления в браузере с зажатым **Shift**.
5. Подожди **5–10 секунд**. Должны подгрузиться скрипт и стили (в Network — статус **200** у `index-….js` и `index-….css`), после чего должен появиться интерфейс приложения.

## Шаг 5. Если по-прежнему Pending или 404 на скрипт

1. В **Network** посмотри точное имя запрашиваемого файла (например `index-dulr0_-s.js`).
2. Открой в **новой вкладке** ссылку на этот файл на домене Vercel (подставь своё имя файла):
   - **https://buddyvpr.vercel.app/assets/ИМЯ_ФАЙЛА.js**
   Если там открывается **код скрипта** — на Vercel файл есть, проблема в домене или кэше. Если там **404** — открой последний деплой на Vercel (вкладка **Deployments** → последний **Ready**), открой **Preview** этого деплоя (ссылка `*.vercel.app`). На странице Preview открой исходный код (ПКМ → «Просмотр кода страницы») и найди строку вида `<script src="/assets/index-….js">`. Скопируй **реальное** имя файла из этой строки.
3. Открой в новой вкладке: **https://www.buddyvpr.ru/assets/РЕАЛЬНОЕ_ИМЯ_ФАЙЛА.js**
   - Если **404** — для домена buddyvpr.ru отдаётся старая версия или кэш; попробуй в Vercel у последнего деплоя нажать **Redeploy** и снова выполнить шаг 4 через 2–3 минуты.
   - Если открывается **скрипт** — значит, с правильным именем файл доступен; тогда снова сделай жёсткое обновление главной страницы (шаг 4), чтобы подтянулся свежий HTML с этим именем.

## Краткий чек-лист
