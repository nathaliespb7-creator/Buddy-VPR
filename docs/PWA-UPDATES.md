# Обновление приложения «добавить на экран» (PWA)

Когда пользователь добавляет Бадди ВПР на домашний экран (iPhone/Android), открывается та же веб-версия, но через иконку и в режиме standalone. Обновления приходят так.

## Как это устроено

1. **При первом открытии** браузер регистрирует Service Worker (`sw.js`), кэширует стартовую страницу, manifest и иконку. Дальше часть запросов идёт из кэша (офлайн или быстрый старт).

2. **При каждом открытии приложения** браузер в фоне проверяет, изменился ли файл `sw.js` на сервере. Если да — скачивает новый worker и устанавливает его. В нашем `sw.js` стоит `skipWaiting()`, поэтому новый worker сразу берёт управление.

3. **Автоперезагрузка:** в коде (см. `client/src/main.tsx`) при смене контроллера (новая версия worker) страница один раз перезагружается. Пользователь получает актуальную версию без ручного «обновить». При самом первом открытии перезагрузки нет.

4. **Деплой на Vercel:** после `git push` и сборки новый `index.html` и JS-чанки попадают на сервер. Файл `sw.js` в `client/public/` тоже деплоится. Если меняли только логику/стили (не имя `sw.js`), браузер увидит изменение при следующей проверке (обычно при следующем открытии приложения или смене вкладки).

## Что видит пользователь

- **Обычно:** открыл приложение с домашнего экрана — через несколько секунд может произойти одна перезагрузка (если вышло обновление), дальше работает новая версия.
- **Если долго не открывал:** при следующем открытии подтянется новая версия и, при необходимости, страница перезагрузится.
- **Офлайн:** показывается последняя закэшированная версия; новые обновления подтянутся, когда появится интернет и пользователь снова откроет приложение.

## Если нужно принудительно сбросить кэш

Пользователь может удалить приложение с домашнего экрана и снова открыть сайт в браузере, затем снова «Добавить на экран» — тогда подтянется свежая версия. Либо в настройках браузера — очистка данных сайта для buddyvpr.vercel.app.

Разработчик при смене стратегии кэша может поменять имя кэша в `client/public/sw.js` (например `vpr-buddy-v2`) — тогда у всех установится новый worker и новый кэш.
