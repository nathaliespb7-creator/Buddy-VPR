## 🔴 Критические ошибки (блокирующие)

### 1. Некорректные ссылки на `ruleId` (ИСПРАВЛЕНО)

| 📍 Файл | Строки | Тип | ❌ Было | ✅ Стало | 💡 Комментарий |
|--------|--------|-----|--------|----------|----------------|
| `client/src/lib/taskData.ts` | ~1178–1336 (множество заданий phonetics) | Критическая | `ruleId: 56` | `ruleId: 2` | В `goldenRules` есть только id 1–22. Правила с id 56 не существует. `getRuleForTask()` возвращал `undefined`, затем fallback по категории (первое правило phonetics). `getBuddyHint(56)` возвращал пустую строку. Для заданий «Сколько звуков/слогов» логически подходит правило id 2 (Непроизносимые согласные, категория phonetics) — общая тема «звуки и буквы». |

**Итог:** Все вхождения `ruleId: 56` заменены на `ruleId: 2`.

## 🟢 Валидация контента (child-friendly)

- **Негативные/пугающие сценарии:** в проверенных заданиях не обнаружены.
- **unacceptablePatterns / unacceptable_situations:** формулировки этически корректны (например, «Когда ситуация поощряет лень в ущерб учёбе» в phraseologyWisdom).
- **Длина вопроса:** часть заданий reading с длинным текстом в `question` может превышать 280 символов — при необходимости можно вынести длинный текст в отдельное поле или сократить для мобильного UI.
- **Количество опций:** у choice-заданий в основном 2–4 варианта — норма.

## 🔧 Структурная целостность

- **Импорты:** в `taskData.ts` импорты `sentences`, `phraseologyWisdomTasks`, `contextIslandTasks` корректны.
- **Экспорты и ID:** `tasksData` собирается из inline-массива + syntaxIslandTasks + phraseologyWisdomTasks + contextIslandTasks; пересечений по `id` нет.
- **START_ID:** phraseologyWisdom 3031, contextIsland 3081 — не пересекаются с основным диапазоном 1–1340 и друг с другом.
- **getRuleForTask, getBuddyHint:** работают с любым `ruleId` из goldenRules; при отсутствии правила — fallback по категории (getRuleForTask) или пустая строка (getBuddyHint). После замены ruleId 56 → 2 подсказки для phonetics заданий отображаются корректно.
