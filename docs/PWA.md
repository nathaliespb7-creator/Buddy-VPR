# PWA: офлайн-режим и установка на устройство

Приложение «Бадди ВПР» настроено как Progressive Web App: его можно установить на телефон или планшет и использовать без интернета после первой загрузки.

## Что сделано

### 1. Манифест (`client/public/manifest.json`)

- **name**: «Бадди ВПР»
- **short_name**: «Бадди»
- **display**: standalone (как отдельное приложение)
- **theme_color**: #4ECDC4
- **background_color**: #F0F9F8
- **icons**: 192×192 и 512×512 (сейчас используется `favicon.png`; при желании замени на отдельные `icon-192.png` и `icon-512.png` в `public/`)

### 2. Плагин Vite PWA (`vite-plugin-pwa`)

- **registerType**: `autoUpdate` — при новой версии приложения SW обновляет кэш в фоне, без принудительной перезагрузки.
- **Кэш статики**: HTML, CSS, JS, шрифты, картинки кэшируются при сборке (Workbox `globPatterns`).
- **Кэш API**: запросы к `/api/*` кэшируются по стратегии **NetworkFirst** (сначала сеть, при отсутствии сети — из кэша). Срок хранения кэша API — 7 дней, до 100 записей.

### 3. iOS

В `client/index.html` добавлены:

- `apple-mobile-web-app-capable`
- `apple-mobile-web-app-status-bar-style`
- `apple-mobile-web-app-title`: «Бадди ВПР»
- `apple-touch-icon`: `/favicon.png`

### 4. Прогресс пользователя

Звёзды и профиль хранятся в **localStorage** (`buddy_profile`, `buddy_session_id`) и не зависят от Service Worker. Они доступны и офлайн.

## Как проверить офлайн-режим (Chrome DevTools)

1. **Собери проект и открой в браузере**
   ```bash
   npm run build:client
   npx serve dist/public
   ```
   Открой указанный адрес (например `http://localhost:3000`).

2. **Проверь, что Service Worker установлен**
   - Открой DevTools (F12).
   - Вкладка **Application** (или **Приложение**).
   - Слева: **Service Workers**. Должен быть зарегистрирован один SW (например от Workbox), статус **activated**.

3. **Проверь кэш**
   - В **Application** → **Cache Storage** — должны быть кэши (например `workbox-precache-...` и при наличии запросов к API — `buddy-api-cache`).

4. **Включи офлайн**
   - В **Application** → **Service Workers** поставь галочку **Offline** (или в **Network** выбери **Offline**).
   - Обнови страницу (F5). Приложение должно открыться из кэша: интерфейс, задания (если уже загружались), прогресс из localStorage.

5. **Выключи офлайн**
   - Сними галочку **Offline**. Работа с сетью снова пойдёт через интернет.

## Установка на устройство

### Android (Chrome)

1. Открой в Chrome сайт приложения (например **https://www.buddyvpr.ru** или твой URL на Vercel).
2. Дождись полной загрузки страницы.
3. Нажми **меню** (три точки справа вверху) → выбери **«Установить приложение»** или **«Добавить на главный экран»**.
4. Подтверди установку. На главном экране появится иконка **«Бадди»**.
5. Чтобы открыть приложение — нажми на эту иконку. Оно откроется в полноэкранном режиме, без адресной строки.

Если пункта «Установить приложение» нет — открой ту же страницу ещё раз или обнови её; иногда браузер показывает подсказку об установке автоматически.

---

### iPhone / iPad (Safari)

1. Открой сайт **в Safari** (не в Chrome или другом браузере): например **https://www.buddyvpr.ru**.
2. Нажми кнопку **«Поделиться»** (квадрат со стрелкой вверх внизу экрана).
3. В меню выбери **«На экран „Домой“»** (или «Add to Home Screen»).
4. При желании измени название (по умолчанию «Бадди ВПР») и нажми **«Добавить»**.
5. На главном экране появится иконка. Чтобы открыть приложение — нажми на неё. Оно откроется в полноэкранном режиме.

На iOS приложение можно ставить только из Safari; из Chrome или Firefox — нельзя.

---

### Компьютер (Chrome или Edge)

- В адресной строке справа может появиться иконка **установки** (дисплей со стрелкой) — нажми её и подтверди установку.
- Или: меню браузера (три точки) → **«Установить Бадди ВПР»** / **«Приложения»** → **«Установить сайт как приложение»**.

После установки приложение открывается в отдельном окне (standalone) и использует те же кэш и данные (localStorage).

## Замена иконок (опционально)

Чтобы использовать отдельные иконки 192×192 и 512×512:

1. Положи в `client/public/` файлы `icon-192.png` и `icon-512.png`.
2. В `client/public/manifest.json` замени пути в `icons`:
   - `"src": "/icon-192.png"`, `"sizes": "192x192"`
   - `"src": "/icon-512.png"`, `"sizes": "512x512"`
3. В `vite.config.ts` в секции `VitePWA` → `manifest` → `icons` укажи те же пути.
4. В `client/index.html` в `<link rel="apple-touch-icon" href="...">` укажи `/icon-192.png`.

После этого пересобери проект и при необходимости очисти кэш браузера.

---

## Сборка

Команда `npm run build:client` генерирует Service Worker (Workbox). Если сборка падает с ошибкой `Unexpected early exit` на этапе записи SW, запусти сборку без песочницы (в обычном терминале): `npm run build:client`. На Vercel сборка выполняется в полноценном окружении и должна проходить успешно.
