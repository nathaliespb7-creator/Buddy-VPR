# Деплой фронтенда на Vercel и привязка домена buddyvpr.ru

## Что сделано в проекте

- Скрипт **`npm run build:client`** — собирает только фронтенд (Vite) в `dist/public`.
- **`vercel.json`** — конфиг для Vercel: команда сборки, папка вывода, SPA-rewrites (все маршруты → `index.html`).
- Переменная **`VITE_API_URL`** — если фронт и бэкенд на разных доменах, в Vercel задаётся URL бэкенда (см. ниже).

---

## Шаг 1: Подключить репозиторий к Vercel

1. Зайди на **https://vercel.com** и войди (через GitHub).
2. **Add New…** → **Project**.
3. **Import** репозитория **Buddy-VPR** (nathaliespb7-creator/Buddy-VPR или твой fork).
4. Vercel подхватит настройки из **`vercel.json`**:
   - **Build Command:** `npm run build:client`
   - **Output Directory:** `dist/public`
   - **Install Command:** `npm install`
5. Поля **Root Directory** не меняй (корень репо).
6. Нажми **Deploy**. Дождись окончания сборки.

После деплоя получишь URL вида `buddy-vpr-xxx.vercel.app`. Проверь, что открывается приложение (без данных, пока бэкенд не подключён — нормально).

---

## Шаг 2: Привязать домен buddyvpr.ru

1. В проекте на Vercel открой вкладку **Settings** → **Domains**.
2. Нажми **Add** и введи **buddyvpr.ru**.
3. Vercel покажет, какие DNS-записи создать у регистратора домена (обычно это A-запись или CNAME).
4. В панели, где куплен домен (reg.ru, timeweb, и т.п.):
   - Добавь запись по инструкции Vercel (часто CNAME: `cname.vercel-dns.com` или A-запись на IP Vercel).
   - При желании добавь **www.buddyvpr.ru** и в Vercel в настройках домена выбери редирект www → без www (или наоборот).
5. Подожди 5–60 минут, пока DNS обновится. В Vercel статус домена станет **Valid** (может быть галочка SSL).
6. Vercel выдаёт SSL (HTTPS) для домена автоматически.

### Шаг 2а: Настройка DNS в nic.ru

Домен **buddyvpr.ru** куплен в nic.ru. Есть **два варианта**.

#### Вариант A: Делегирование на DNS Vercel (рекомендуется)

1. Зайди на **https://www.nic.ru** → **Мои домены** → **buddyvpr.ru**.
2. Открой раздел **DNS-серверы домена** (или **Указать DNS-серверы**).
3. Выбери **«Указать DNS-серверы самостоятельно»** и укажи:
   - **ns1.vercel-dns.com**
   - **ns2.vercel-dns.com**
4. Сохрани. **В nic.ru больше ничего добавлять не нужно** — зоной управляет Vercel, записи A/CNAME создаются автоматически, когда домен добавлен в проект (Settings → Domains).
5. Подожди распространения DNS (от 15 минут до 24–48 часов).

#### Вариант B: Зона остаётся в nic.ru (A + CNAME)

Если не используешь NS Vercel, а оставляешь DNS в nic.ru:

1. **Мои домены** → **buddyvpr.ru** → **Управление зоной** (или **Редактировать зону**).
2. Добавь записи:
   - **A** — имя `@`, значение **76.76.21.21**
   - **CNAME** — имя **www**, значение **cname.vercel-dns.com**
3. Сохрани и подожди обновления DNS.

---

В обоих случаях после настройки в Vercel во вкладке Domains статус домена должен стать **Valid**. Если Vercel при добавлении домена показал другие значения — используй их.

#### Если сайт всё ещё не открывается (DNS_PROBE_FINISHED_NXDOMAIN)

Попробуй **Вариант B**: перестать использовать NS Vercel и настроить зону в nic.ru.

1. **В nic.ru** → **Мои домены** → **buddyvpr.ru**.
2. **DNS-серверы домена**: выбери **«DNS-серверы услуг Руцентр»** → **«DNS-master»** (или **«Хостинг»**, если используешь хостинг Руцентр). Так зоной снова будет управлять nic.ru.
3. Открой **«Управление зоной»** / **«Редактировать зону»** для **buddyvpr.ru**.
4. Удали старые записи для `@` и `www`, если они мешают. Добавь **ровно две** записи:

   | Тип    | Имя (поддомен) | Значение / Хост              |
   |--------|----------------|-------------------------------|
   | **A**  | `@` (или пусто) | **76.76.21.21**               |
   | **CNAME** | **www**      | **cname.vercel-dns.com**      |

5. Сохрани. Подожди 15–60 минут, проверь **https://buddyvpr.ru** и **https://www.buddyvpr.ru**.

Домен в Vercel (Settings → Domains) не трогай — он уже добавлен. После появления записей в nic.ru резолв подтянется и сайт начнёт открываться.

## Шаг 3 (позже): Подключить бэкенд

Сейчас фронт на Vercel **не имеет бэкенда**: запросы к `/api/*` будут падать (или приложение будет использовать только локальные данные, если есть fallback).

Когда захочешь подключить API:

1. Разверни бэкенд (Express + PostgreSQL) на любом хостинге (Railway, Render, Yandex Cloud, и т.д.).
2. Получи URL бэкенда, например `https://buddy-vpr-api.railway.app`.
3. В Vercel: **Settings** → **Environment Variables** → добавь переменную:
   - **Name:** `VITE_API_URL`
   - **Value:** `https://buddy-vpr-api.railway.app` (без слэша в конце)
   - Включи для **Production** (и при необходимости Preview).
4. Сделай **Redeploy** проекта на Vercel (чтобы сборка подхватила новую переменную).
5. На бэкенде включи **CORS** для домена `https://buddyvpr.ru` (и при необходимости для `https://*.vercel.app`).

После этого фронт на buddyvpr.ru будет ходить в API по указанному URL.

## Сайт не открывается — что проверить

### 1. Код должен быть на GitHub

Vercel собирает проект из репозитория. Если последние изменения не запушены, на сайте будет старая (или пустая) версия.

- В терминале: `cd /Users/nathalie/Buddy-VPR` → `git status` → если есть незапушенные коммиты, выполни `git push origin main`.
- Инструкция по токену и push: **docs/GIT-GITHUB.md**.

### 2. Деплой на Vercel

- Зайди на **https://vercel.com** → свой проект **Buddy-VPR**.
- Вкладка **Deployments**: последний деплой должен быть в статусе **Ready** (зелёная галочка). Если **Failed** — открой деплой и посмотри логи ошибки.
- Если пушила только что — подожди 1–2 минуты и обнови страницу деплоев.

### 3. Домен и DNS

- **Settings** → **Domains**: для **buddyvpr.ru** и **www.buddyvpr.ru** статус должен быть **Valid**. Если **Invalid** или **Pending** — проверь записи в nic.ru (см. шаг 2а выше).
- Проверь с телефона с мобильным интернетом или в другом браузере: иногда мешает кэш или локальный DNS.

### 4. Страница скачивается вместо отображения (Safari, Яндекс.Браузер)

В проекте включён **Edge Middleware** (`middleware.js` в корне): для запросов к `/` и `/index.html` он задаёт заголовки `Content-Type: text/html; charset=utf-8` и `X-Content-Type-Options: nosniff`, запрос передаётся дальше без изменения тела ответа. Так Safari и Яндекс.Браузер должны открывать страницу, а не предлагать сохранить файл.

- Зависимость **`@vercel/functions`** должна быть в `package.json` (devDependencies) — без неё middleware не заработает.
- После правок сделай **push** и дождись деплоя на Vercel, затем проверь **https://www.buddyvpr.ru** в Safari/Яндексе.
- Если проблема остаётся: открой сайт в **Chrome** для проверки, попробуй **https://www.buddyvpr.ru/index.html** или очисти кэш браузера.

### 5. Ошибка ERR_NAME_NOT_RESOLVED / «Не удаётся получить доступ к сайту» / NXDOMAIN

Домен **не резолвится** в IP — браузер не знает, куда идти. Исправляется только настройкой DNS у регистратора (nic.ru).

**Что сделать:**

1. **Проверь, что вводишь с www:** открой именно **https://www.buddyvpr.ru** (не только buddyvpr.ru). Иногда настроена только запись для www.
2. **Зайди в nic.ru** → **Мои домены** → **buddyvpr.ru**.
3. **Убедись, что зоной управляет nic.ru:** в разделе **«DNS-серверы домена»** должно быть **«DNS-серверы услуг Руцентр»** / **«DNS-master»**, а не ns1.vercel-dns.com (если стоят NS Vercel, но записей в Vercel нет — домен не резолвится).
4. Открой **«Управление зоной»** (или **«Редактировать зону»**). Должны быть **две** записи:
   - **A** — имя `@` (или пусто) — значение **76.76.21.21**
   - **CNAME** — имя **www** — значение **cname.vercel-dns.com.** (в nic.ru часто нужна **точка в конце**, иначе получится www.buddyvpr.ru.buddyvpr.ru)
5. Сохрани. Подожди **15–60 минут** (иногда до нескольких часов). Проверь снова **https://www.buddyvpr.ru** и **https://buddyvpr.ru**.
6. В **Vercel** → проект → **Settings** → **Domains**: домены buddyvpr.ru и www.buddyvpr.ru должны быть добавлены; после обновления DNS статус станет **Valid**.
