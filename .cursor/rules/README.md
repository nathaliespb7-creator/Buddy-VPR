# Правила Cursor для Бадди ВПР

Здесь лежат правила (`.mdc`), которые Cursor подставляет в контекст при работе с проектом.

## Текущие файлы

| Файл | Назначение |
|------|------------|
| `buddy-vpr-role.mdc` | Роль, стиль общения, требования и запреты (всегда) |
| `buddy-vpr-product.mdc` | Продукт, аудитория, фичи, приоритеты (всегда) |
| `buddy-vpr-tech.mdc` | Стек и конвенции кода (при открытии .ts/.tsx) |
| `buddy-vpr-offline-security.mdc` | Офлайн/PWA и 152-ФЗ (клиент, сервер, shared) |
| `buddy-vpr-code-style.mdc` | Стиль кода, нейминг, обработка ошибок |
| `buddy-vpr-qa.mdc` | QA-агент: при «проверь проект» / «аудит» / «перед деплоем» — чек-лист из QA-AGENT.md |

## Как добавить новое правило

1. Создай файл с суффиксом `.mdc` в этой папке.
2. В начале файла укажи frontmatter:

```yaml
---
description: Краткое описание правила
alwaysApply: true   # или false
globs: "**/*.tsx"   # если alwaysApply: false — для каких файлов
---
```

3. Ниже пиши заголовки и текст (лаконично, до ~50 строк на правило).
