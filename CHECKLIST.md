# Чек-лист успешного открытия Buddy-VPR

## Подготовка окружения

- [ ] Cursor открыт
- [ ] Репозиторий склонирован на Mac
- [ ] Папка проекта открыта в Cursor
- [ ] Файлы видны в левой панели
- [ ] Терминал работает (`⌃` + `` ` ``)

## Установка и запуск

- [ ] `npm install` прошёл без ошибок
- [ ] Файл `.env` создан с `DATABASE_URL` и `PORT`
- [ ] `npm run db:push` выполнен (схема БД применена)
- [ ] Проект запускается локально (`npm run dev`)

## Проверка

- [ ] В терминале видно `serving on port 5000` и ссылки
- [ ] Приложение открывается в браузере: **http://127.0.0.1:5000** или **http://localhost:5000**
- [ ] Splash screen и главный экран отображаются корректно

## Если ничего не открывается

1. **Сервер запущен?** В терминале должна быть строка `serving on port 5000`. Если нет — запустите `npm run dev` из папки проекта.
2. **Копируйте ссылку из терминала** — после запуска появятся строки с http://127.0.0.1:5000 и http://localhost:5000. Вставьте одну из них в адресную строку браузера.
3. **Пробуйте оба адреса:** сначала http://127.0.0.1:5000, если не открылось — http://localhost:5000.
4. **Другой браузер:** Safari, Chrome или Firefox.
5. **Сайт в интернете:** https://buddyvpr.ru или https://buddyvpr.vercel.app — если домены настроены, сайт должен открываться там.
